## Overview

LaTeX-based, JSON-driven, version-tracked, ATS-friendly CV for Computer Science / AI / Software Engineering roles,
with automated GitHub Actions build and public release publishing.

All CV content lives in `data.json` (following the [JSON Resume schema](https://jsonresume.org/schema/)).
A Python + Jinja2 renderer generates `cv.tex` from the JSON, which is then compiled to PDF by `latexmk`.

## Repository Structure

```
jsurrea/CV/
├── data.json              # Single source of truth — all CV content
├── template/
│   └── cv.tex.j2          # Jinja2 LaTeX template (the visual design)
├── generate.py            # Reads data.json + renders template → cv.tex, then compiles PDF
├── .github/
│   └── workflows/
│       └── build.yml      # Manual-trigger workflow: generate → compile → release
├── cv.tex                 # (gitignored) generated by running generate.py
├── cv.pdf                 # (gitignored) produced by CI and attached to Release
├── .gitignore
└── README.md
```

**Key decisions:**
- `data.json` is committed and always current — it's the source of truth and publicly readable at  
  `https://raw.githubusercontent.com/jsurrea/CV/main/data.json`.
- `cv.tex` is **generated** (not hand-written) and committed so it's diffable.
- `cv.pdf` is **not committed** to `main` — it is attached as an asset to a GitHub Release.
- Each workflow run creates a new Release tagged with the ISO date (e.g., `v2026.03.05`).

## Prerequisites

- Python 3.9+
- TeX Live (Full or Extras) — or MiKTeX on Windows
  - Required packages: `latexmk`, `texlive-latex-extra`, `texlive-fonts-extra`
- `pip install jinja2`

## Local Build

```bash
git clone https://github.com/jsurrea/CV.git
cd CV

# Install Python dependency
pip install jinja2

# Full build (generates cv.tex then cv.pdf)
python generate.py

# Open the result
open cv.pdf          # macOS
xdg-open cv.pdf      # Linux
```

## CLI Options

```bash
python generate.py              # Full build: generate .tex + compile .pdf
python generate.py --tex-only  # Generate cv.tex only (for inspection/debugging)
python generate.py --pdf-only  # Compile pre-existing cv.tex only
```

## Editing Your CV

1. Open `data.json` and modify the relevant section (`work`, `education`, `projects`, etc.)
2. Run `python generate.py` to regenerate the PDF
3. Commit changes to `data.json` (and optionally `cv.tex`)
4. Trigger the GitHub Actions workflow to publish a new versioned Release

## Publishing a New Version

Go to **Actions → Build CV → Run workflow** in the GitHub repository UI,
optionally specifying a version tag (defaults to today's date in `v2026.03.05` format).
